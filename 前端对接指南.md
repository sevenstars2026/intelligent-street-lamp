## 🚀 在你的机器上启动后端

### 1. 环境要求
- Node.js >= 14.0
- npm >= 6.0

### 2. 安装依赖
```bash
cd task3-backend
npm install
```

### 3. 启动服务
```bash
npm run dev
```

### 4. 验证服务
```bash
# 打开浏览器访问：
http://localhost:3000/api/health

# 或使用curl
curl http://localhost:3000/api/health
```

**看到以下输出表示成功：**
```json

  "code": 200,
  "message": "healthy",
  "data": {
    "status": "up",
    "services": {
      "database": "up",
      "redis": "up",
      "mqtt": "up"

  }
}
```

## 🎯 可用接口列表

### 设备控制
```bash
# 控制单个路灯
POST http://localhost:3000/api/devices/lamp_001/control
Body: {"command": "on"}

# 批量控制
POST http://localhost:3000/api/devices/batch-control
Body: {"deviceIds": ["lamp_001", "lamp_002"], "command": "off"}

# 控制历史
GET http://localhost:3000/api/devices/lamp_001/control-logs?page=1&pageSize=20
```

### 阈值管理
```bash
# 获取阈值
GET http://localhost:3000/api/devices/lamp_001/threshold

# 设置阈值
POST http://localhost:3000/api/devices/lamp_001/threshold
Body: {"lightThresholdOn": 200, "lightThresholdOff": 300}
```

### 设备模式
```bash
# 获取模式
GET http://localhost:3000/api/devices/lamp_001/mode

# 切换模式
PUT http://localhost:3000/api/devices/lamp_001/mode
Body: {"mode": "auto"}
```

### 告警管理
```bash
# 告警列表
GET http://localhost:3000/api/alarms?status=active

# 告警详情
GET http://localhost:3000/api/alarms/1

# 标记已处理
PUT http://localhost:3000/api/alarms/1/resolve
Body: {"note": "已处理"}
```

## 💡 前端开发示例

### React示例
```javascript
// 配置API基础地址
const API_BASE_URL = 'http://localhost:3000/api';

// 控制路灯
async function controlLight(deviceId, command) {
  const response = await fetch(`${API_BASE_URL}/devices/${deviceId}/control`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ command })
  });
  
  const data = await response.json();
  return data;
}

// 使用
controlLight('lamp_001', 'on').then(result => {
  console.log('控制结果:', result);
});
```

### Vue示例
```javascript
// 使用axios
import axios from 'axios';

const api = axios.create({
  baseURL: 'http://localhost:3000/api'
});

// 控制路灯
export const controlDevice = (deviceId, command) => {
  return api.post(`/devices/${deviceId}/control`, { command });
};

// 获取告警列表
export const getAlarms = (params) => {
  return api.get('/alarms', { params });
};
```

## 🔧 常见问题

### Q1: 启动失败怎么办？
```bash
# 检查端口是否被占用
lsof -i :3000

# 如果被占用，杀死进程或修改端口
# 修改 .env 文件：
PORT=3001
```

### Q2: 跨域问题怎么解决？
后端已配置CORS，允许所有来源。如果仍有问题，检查前端请求是否正确。

### Q3: Mock数据在哪里？
启动服务时会自动创建3个测试设备：
- lamp_001
- lamp_002
- lamp_003

### Q4: 需要认证吗？
目前Mock了认证，所有请求都会当作已登录用户处理。真实认证需要等任务2完成。
