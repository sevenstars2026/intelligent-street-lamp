### 2.1 MQTT消息接口规范
**状态：** 需要任务2实现完成后通知我

```
后端→硬件（控制指令下发）：
- MQTT Topic: devices/{deviceId}/control
- 消息格式示例：
{
  "cmd": "switch",           // 命令类型
  "value": "on",             // 参数值（on/off）
  "requestId": "req_xxx",    // 请求ID，用于匹配反馈
  "timestamp": 1719734400000
}

硬件→后端（控制结果反馈）：
- MQTT Topic: devices/{deviceId}/control/response
- 消息格式示例：
{
  "requestId": "req_xxx",        // 对应的请求ID
  "status": "success",           // success/failed
  "result": "on",                // 实际执行结果
  "message": "开灯成功",
  "timestamp": 1719734400000
}

光照数据上报：
- MQTT Topic: devices/{deviceId}/data
- 消息格式示例：
{
  "deviceId": "lamp_001",
  "lightIntensity": 250,  // 光照强度数值
  "status": "on",         // 当前状态
  "timestamp": 1719734400000
}

心跳包：
- MQTT Topic: devices/{deviceId}/heartbeat
- 消息格式示例：
{
  "deviceId": "lamp_001",
  "timestamp": 1719734400000
}

需要明确的具体问题：
- [ ] MQTT Broker的连接地址是什么？（IP:Port）
- [ ] MQTT Broker使用什么认证方式？（用户名/密码？证书？无认证？）
- [ ] 后端订阅的Topic列表有哪些？
- [ ] 后端发布的Topic列表有哪些？
- [ ] 光照强度的取值范围具体是多少？（比如：0-4095 for 12bit ADC）
- [ ] 光照数据上报频率具体是多少秒？（比如：每5秒上报一次）
- [ ] 心跳包发送频率具体是多少秒？（比如：每30秒发送一次）
- [ ] 设备ID的具体格式是什么？（比如：lamp_001, lamp_002 这样的格式）
```
