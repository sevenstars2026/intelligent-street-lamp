# 智慧路灯系统 - 任务分解（技术架构视角）

## 任务1：硬件端开发
**负责团队：** 硬件/嵌入式团队

### 开发内容
- **传感器模块**
  - 光照强度传感器集成
  - 数据采集与预处理
  - 采样频率控制

- **控制模块**
  - 路灯开关继电器控制
  - 指令接收与执行
  - 执行结果反馈

- **通信模块**
  - MQTT客户端实现
  - 心跳包定时发送
  - 断线重连机制
  - 数据上报格式定义

- **设备管理**
  - 设备唯一标识（deviceId）
  - 参数配置存储
  - 固件升级接口

### 技术栈
- 嵌入式C/C++
- ESP32/STM32等MCU
- MQTT协议
- JSON数据格式

### 交付物
- 硬件设备固件代码
- MQTT消息格式文档
- 设备调试工具
- 硬件接口说明文档

### 预估工作量
10-12人天

## 团队协作接口定义

### 硬件 ↔ 后端
**MQTT Topic规范**
```
上行（硬件→后端）：
- devices/{deviceId}/data        # 传感器数据上报
- devices/{deviceId}/heartbeat   # 心跳包
- devices/{deviceId}/status      # 状态反馈

下行（后端→硬件）：
- devices/{deviceId}/control     # 控制指令
- devices/{deviceId}/config      # 参数配置
```

**消息格式示例**
```json
// 光照数据上报
{
  "deviceId": "lamp_001",
  "timestamp": 1719734400000,
  "lightIntensity": 250,
  "status": "on"
}

// 控制指令
{
  "cmd": "switch",
  "value": "on",
  "timestamp": 1719734400000
}
```

### 后端 ↔ 前端
**RESTful API规范**
```
GET    /api/devices                  # 获取设备列表
POST   /api/devices                  # 添加设备
GET    /api/devices/{id}/data        # 获取设备实时数据
POST   /api/devices/{id}/control     # 控制设备
GET    /api/light/history            # 历史光照数据
GET    /api/alarms                   # 告警列表
POST   /api/chat                     # 智能问答
```

**WebSocket推送**
```
ws://backend/realtime
- 实时光照数据推送
- 设备状态变更通知
- 告警实时推送
```
