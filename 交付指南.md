#### 📍 发什么给他们
```

   ├─ SUMMARY.md
   └─ 任务3_API接口设计文档.md（查看MQTT部分）

✅ 告诉他们：
   "按照文档中的Topic和消息格式规范"
```

#### 🎯 他们需要完成什么

**硬件需要实现的MQTT通信：**

1. **发送心跳** ✅ 必需
   ```
   Topic: devices/{deviceId}/heartbeat
   频率: 每30秒
   内容: {deviceId, timestamp}
   ```

2. **上报光照数据** ✅ 必需
   ```
   Topic: devices/{deviceId}/data
   频率: 每5分钟或有变化时
   内容: {deviceId, lightIntensity, status, timestamp}
   ```

3. **监听控制指令** ✅ 必需
   ```
   Topic: devices/{deviceId}/control
   解析: {cmd, value, requestId}
   value: "on" 或 "off"
   ```

4. **反馈控制结果** ✅ 必需
   ```
   Topic: devices/{deviceId}/control/response
   反馈: {requestId, status, result, message, timestamp}
   status: "success" 或 "failed"
   ```

#### 📝 硬件的工作清单
```
□ 实现MQTT连接
□ 按规范发送心跳（devices/{deviceId}/heartbeat）
□ 按规范上报光照数据（devices/{deviceId}/data）
□ 监听控制指令（devices/{deviceId}/control）
□ 按规范反馈控制结果（devices/{deviceId}/control/response）

这些完成后，可以与任务3的Mock MQTT进行对接测试
```
