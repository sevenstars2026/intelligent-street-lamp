## 二、路由设计

| 路径 | 组件 | 说明 |
|---|---|---|
| `/` | `MapPage.vue` | **主页**：地图 + 推荐弹窗 + 熄灯提醒 |
| `/routes` | `RoutesPage.vue` | 游览路线列表 |
| `/spots` | `SpotsPage.vue` | 拍照点列表 |
| `/events` | `EventsPage.vue` | 表演活动列表 |
| `/report` | `ReportPage.vue` | 故障上报表单 |

**无登录守卫**：所有路由公开访问。上报时在表单内填写姓名和手机号。

**URL 参数解析**（QR 码入口）：
```
/?lamp=lamp_003
```
- `MapPage.vue` 的 `onMounted` 中检查 `route.query.lamp`
- 如果有值，自动将该路灯设为"当前选中"，弹出推荐弹窗
- 游客可关闭弹窗返回地图自由浏览

**表单校验**：
- 姓名：必填，2-20 字
- 手机号：必填，11 位数字格式
- 路灯编号：必填，格式 `lamp_XXX`
- 故障描述：必填，10-200 字

**提交流程**：
1. 构建 `FormData`（含文本字段 + 图片文件）

3. 成功 → Toast "上报成功，工作人员将尽快处理" + 清空表单
4. 失败 → Toast 提示错误信息

## 四、API 层 (`src/utils/api.ts`)

在管理端 `api.ts` 基础上新增以下函数：

```javascript
import axios from 'axios';

const api = axios.create({
  baseURL: '/api',
  timeout: 10000,
});

// 响应拦截器（与管理端一致）
api.interceptors.response.use(
  res => {
    const { code, message, data } = res.data;
    if (code !== 200) return Promise.reject(new Error(message));
    return data;
  },
  err => Promise.reject(err)
);

// ===== 已有函数（复用） =====
// getDevices(), getDeviceById(), 等...

// ===== 新增景区函数 =====

// 获取游览路线
export function getScenicRoutes() {
  return api.get('/scenic/routes');
}

// 获取拍照点
export function getScenicSpots() {
  return api.get('/scenic/spots');
}

// 获取表演活动
export function getScenicEvents() {
  return api.get('/scenic/events');
}

// 获取路灯位置（SVG 坐标）
export function getScenicLamps() {
  return api.get('/scenic/lamps');
}

// 故障上报（FormData 含图片）
export function submitFaultReport(formData) {
  return api.post('/reports', formData, {
    headers: { 'Content-Type': 'multipart/form-data' },
    timeout: 30000,  // 图片上传可能较慢
  });
}
```

## 五、核心组件设计

### 5.1 ScenicMap.vue — 景区地图（核心组件）

```
Props: 无
State: lamps, routes, spots, events, activeFilter, selectedLamp
```

**底图方案**：AI 生成的景区俯瞰插画（PNG/WebP），放在 `assets/map.webp`，作为背景图。

**标记方案**：路灯/拍照点/活动用 CSS 绝对定位 `<div>` 叠加在底图上，坐标从 API 获取（百分比）。

**缩放移动方案**：使用 `panzoom` 轻量库（9KB），整个地图（底图+标记）包在一个内层容器中，缩放移动作用于整个容器。

```html
<!-- 结构示意 -->
<div class="map-viewport">           <!-- overflow: hidden，手机屏幕大小 -->
  <div class="map-inner" ref="mapEl">  <!-- panzoom 操作的目标 -->
    <img src="@/assets/map.webp" />    <!-- AI 底图，2x 分辨率 -->
    <div class="marker lamp" v-for="l in lamps"
         :style="{ left: l.x + '%', top: l.y + '%' }" />    <!-- 路灯标记 -->
    <div class="marker spot" v-for="s in spots" ... />      <!-- 拍照点标记 -->
    <div class="marker event" v-for="e in events" ... />    <!-- 活动标记 -->
  </div>
</div>
```

**panzoom 集成**：
```javascript
import panzoom from 'panzoom'

const mapEl = ref(null)
let pz = null

onMounted(() => {
  pz = panzoom(mapEl.value, {
    maxZoom: 3,          // 最大放大 3 倍
    minZoom: 0.5,        // 最小缩小到 50%
    bounds: true,        // 不拖出边界
    boundsPadding: 0.2,  // 边界外可拖 20%
  })
  // 双击恢复初始视图
  mapEl.value.addEventListener('dblclick', () => {
    pz.moveTo(0, 0)
    pz.zoomAbs(0, 0, 1)
  })
})

onUnmounted(() => pz?.dispose())
```

**底图分辨率要求**：至少 780×1688（390×844 的 2x），放大 3 倍不模糊。AI 生图时指定高分辨率输出。

**筛选逻辑**：
```javascript
const visibleMarkers = computed(() => {
  if (activeFilter.value === 'all') return { lamps, spots, events };
  if (activeFilter.value === 'spots') return { spots };
  // ...
});
```

### 5.2 LampReminder.vue — 熄灯倒计时

```javascript
// 从设备列表中找到自动模式的熄灯时间
// 如果设备模式有定时关闭规则，计算剩余时间
// 否则使用默认值（如 22:00）
const minutesLeft = computed(() => {
  const now = new Date();
  const offTime = new Date();
  offTime.setHours(22, 0, 0, 0);  // 默认 22:00 熄灯
  const diff = offTime - now;
  return Math.max(0, Math.floor(diff / 60000));
});
```

**显示逻辑**：剩余 0 分钟时隐藏。剩余 ≤ 30 分钟时切换为警告样式（橙色/红色闪烁）。

### 5.3 PhotoUploader.vue — 图片上传

- 支持点击选择 + 拍照（`capture="environment"` 移动端后置摄像头）
- 预览缩略图
- 单张删除
- 最多 3 张，超出时隐藏上传按钮
- 文件大小校验：单张 ≤ 5MB，超出时 Toast 提示

## 六、设计系统定位

游客端 UI 风格建议：**暖色调景区风**，区别于管理端的暗色科技风。

| 属性 | 管理端 | 游客端 |
|---|---|---|
| 主色调 | `#658ae4` 科技蓝 | `#f0a050` 暖橙金 |
| 背景 | `#222b45` 深蓝黑 | `#fefaf6` 米白暖 |
| 卡片 | `glass-card`（半透明玻璃） | 圆角白卡 + 微阴影 |
| 字体 | Space Grotesk / Fira Code | 系统字体优先（移动端渲染快） |
| 圆角 | 中等（8px） | 大圆角（12-16px） |
| 图标风格 | 线性图标 | emoji 优先（降低加载成本） |

**CSS 变量**（游客端 `style.css`）：

## 七、QR 码方案

### 7.1 QR 码内容格式

每个路灯的 QR 码编码内容：
```
https://景区域名/?lamp=lamp_003
```

### 7.2 前端解析

```javascript
// 在 MapPage.vue 的 setup 中
import { useRoute } from 'vue-router';

const route = useRoute();

onMounted(() => {
  const lampId = route.query.lamp;
  if (lampId) {
    // 1. 验证 lampId 是否存在于灯具列表中
    // 2. 将地图中心定位到该路灯
    // 3. 自动弹出 RecommendCard
    selectLamp(lampId);
    showRecommendPopup.value = true;
  }
});
```

### 7.3 手动输入（备选入口）

地图页顶部搜索栏，输入路灯编号 → 回车 → 同上述定位逻辑。

### 7.4 QR 码生成

QR 码图片不在本项目生成。管理端后续可加"导出路灯 QR 码"功能，使用 `qrcode` npm 包生成 PNG，管理员打印张贴到路灯上。5 天内不强制实现，可手动用在线工具生成几张演示用。
