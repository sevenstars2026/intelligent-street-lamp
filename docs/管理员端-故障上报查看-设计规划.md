## 一、背景与现状

游客端（`tourist-frontend`）已实现故障上报功能（`POST /api/reports`），但上报数据处于孤岛状态：

| 现状 | 问题 |
|---|---|
| 提交走 `POST /api/reports` | ✅ 正常 |
| 数据存入 `fault_reports` 表 | ✅ 正常 |
| 同时写入 `alarms` 表（`alarmType: 'fault_report'`） | ❌ 未实现 — `alarmId` 硬编码为 `0` |
| `Alarm` 类型定义包含 `'fault_report'` | ❌ 类型联合中缺失 |
| 管理员查看上报的接口 | ❌ 不存在 — 无 `GET /api/reports` |
| 管理端展示页面 | ❌ 不存在 |

**结论**：游客能提交，管理员看不到。

---

## 二、实现策略

### 策略选择：两路并存

既不破坏现有告警体系，又能独立展示故障上报的完整信息（含现场照片）：

```
游客提交 → POST /api/reports
           ├── ① 写入 fault_reports 表（含照片名、上报人联系方式）
           └── ② 同步写入 alarms 表（alarmType: 'fault_report'，关联 lampId）

管理员 → GET /api/alarms        → 告警列表中能看到"游客上报"类型（概要）
       → GET /api/reports       → 故障上报专属页面（含照片、联系方式详情）
       → PUT /api/reports/:id/resolve → 处理上报（联动解决对应的 alarm）
```

export interface FaultReportDetail {

### 3.2 新增 API 接口（3 个）

#### GET /api/reports

查询故障上报列表，支持筛选和分页。

**Query Parameters**：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| status | string | ❌ | `active` / `resolved` |
| lampId | string | ❌ | 按路灯编号筛选 |
| page | number | ❌ | 页码，默认 1 |
| pageSize | number | ❌ | 每页条数，默认 20 |

**Response 200**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "reports": [
      {
        "id": 1,
        "alarmId": 42,
        "reporterName": "张三",
        "reporterPhone": "13800138000",
        "lampId": "lamp_003",
        "description": "路灯闪烁严重，影响夜间拍照",
        "photoUrls": ["1720500000-123456789.jpg"],
        "status": "active",
        "createdAt": "2026-07-09T18:30:00+08:00",
        "resolvedAt": null
      }
    ],
    "pagination": {
      "page": 1, "pageSize": 20, "total": 5, "totalPages": 1
    }
  }
}
```

#### GET /api/reports/:id

获取单条上报详情（含完整照片 URL）。

```json
{
  "code": 200,
  "message": "success",
  "data": { /* FaultReportDetail 对象 */ }
}
```

#### PUT /api/reports/:id/resolve

管理员处理后标记为已解决。

**Request Body**：

```json
{ "note": "已安排维修人员前往处理" }
```

**Response 200**：

```json
{
  "code": 200,
  "message": "已标记为处理完成",
  "data": {
    "id": 1,
    "resolvedAt": "2026-07-09T20:00:00+08:00",
    "note": "已安排维修人员前往处理"
  }
}
```

### 3.3 改造 POST /api/reports（闭环）

修改 [report.service.ts:11](task3-backend/src/services/report.service.ts#L11)，提交时同步创建 Alarm：

```
ReportService.submit()
  ├── ① DatabaseService.addAlarm({ alarmType: 'fault_report', alarmLevel: 'medium', ... })
  │        → 获取 alarmId
  └── ② DatabaseService.addFaultReport({ alarmId, ... })
```

这样管理员既能在告警列表看到，也能在故障上报页面看详情。

## 四、管理端前端设计

### 4.1 页面设计 — FaultReportsPage.vue

复用现有 `AlarmPage.vue` 的 UI 模式（暗色主题、glass-card、骨架屏、分页）。

**布局**（从上到下）：

```
┌─────────────────────────────────────────────┐
│  故障上报                                    │  ← 工具栏
│  [ 全部状态 ▼ ] [ 路灯编号筛选 ▼ ]  [ 刷新 ]   │
├─────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────┐│
│  │ ID │ 上报人 │ 手机号 │ 路灯 │ 描述 │ 照片 │ 时间 │ 状态 │ 操作 │  ← 表格
│  ├─────────────────────────────────────────┤│
│  │ 1  │ 张三   │ 138*** │ lamp_003 │ 路灯闪 │ [图] │ 07-09 18:30 │ 待处理 │ [处理] │
│  │ 2  │ 李四   │ 139*** │ lamp_001 │ 灯不亮 │ [图] │ 07-09 19:00 │ 待处理 │ [处理] │
│  └─────────────────────────────────────────┘│
│                          第 1 / 1 页        │  ← 分页
└─────────────────────────────────────────────┘
```

**表格列设计**：

| 列 | 字段 | 说明 |
|---|---|---|
| ID | `id` | 编号，等宽字体 |
| 上报人 | `reporterName` | — |
| 手机号 | `reporterPhone` | 脱敏显示：`138****8000` |
| 故障路灯 | `lampId` | 路灯编号 |
| 描述 | `description` | 截断显示，hover/点击看全文 |
| 现场照片 | `photoUrls` | 缩略图（最多展示 3 张），点击放大 |
| 上报时间 | `createdAt` | 格式 `MM-DD HH:mm` |
| 状态 | `status` | `active` → "待处理" 橙色 / `resolved` → "已处理" 绿色 |
| 操作 | — | 待处理时显示 [处理] 按钮，已处理显示处理时间 |

**交互行为**：
- 点击照片缩略图 → 弹出大图预览（复用 `ModalOverlay.vue` 模式）
- 点击描述 → 展开全文 tooltip
- [处理] 按钮 → confirm 弹窗确认 → `PUT /api/reports/:id/resolve` → 刷新列表

### 4.4 API 函数（[api.ts](task4-frontend/src/utils/api.ts)）

```typescript
// ===== 故障上报 =====
export function getFaultReports(params?: {
  status?: 'active' | 'resolved'
  lampId?: string
  page?: number
  pageSize?: number
}) {
  return api.get('/reports', { params })
}

export function getFaultReport(id: number) {
  return api.get(`/reports/${id}`)
}

export function resolveFaultReport(id: number, note?: string) {
  return api.put(`/reports/${id}/resolve`, { note })
}
```

### 4.6 照片查看方案

有 3 种方案：

| 方案 | 描述 | 复杂度 |
|---|---|---|
| A | 点击缩略图 → 复用现有 `ModalOverlay.vue` 弹窗放大 | 低 |
| B | 点击缩略图 → 新标签页打开图片 URL | 最低 |
| C | 集成图片 lightbox 库 | 高 |

**推荐方案 A**：管理端已有 `ModalOverlay.vue` 组件，在表格中显示小缩略图，点击时用 Modal 展示大图。照片 URL 通过 `/uploads/{filename}` 访问（后端已有的静态文件服务）。
