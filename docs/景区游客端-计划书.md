# 景区智慧路灯 — 游客端计划书

> 版本：v1.0 | 日期：2026-07-08 | 状态：✅ 已实施

---

## 一、项目背景

智慧路灯项目已完成为市政/管理员使用的管理端（`task4-frontend`）和后端 API 服务（`task3-backend`）。现计划新增面向景区游客的独立前端应用，将产品从"路灯管控工具"升级为"景区智慧路灯系统"。

## 二、核心定位

| 维度 | 管理端（已有） | 游客端（新增） |
|---|---|---|
| 用户 | 市政人员 / 景区管理员 | 景区游客 |
| 入口 | PC 浏览器 | 手机扫码（路灯 QR 码） |
| 核心任务 | 监控路灯、处理告警、设备控制 | 发现拍照点、查看表演活动、导航导览、故障上报 |
| 技术栈 | Vue 3 SPA (`task4-frontend/`) | 独立 Vue 3 SPA (`tourist-frontend/`) |
| 端口 | :5173 | :5174 |

## 三、架构设计

```
                     ┌─────────────────────────┐
                     │    Express 后端 :3000     │
                     │    (task3-backend)        │
                     │                          │
                     │  · 设备管理 11 个 API     │
                     │  · 景区数据  3 个 API ← 新增 │
                     │  · 故障上报  1 个 API ← 新增 │
                     │  · 图片上传  1 个路由 ← 新增 │
                     │  · 告警系统（已有）        │
                     └──────┬───────┬──────────┘
                            │       │
              ┌─────────────┘       └─────────────┐
              ▼                                   ▼
   ┌──────────────────┐              ┌──────────────────┐
   │ task4-frontend   │              │ tourist-frontend │
   │ 管理端 (PC)      │              │ 游客端 (手机)     │
   │ :5173            │              │ :5174            │
   │                  │              │                  │
   │ · 设备仪表盘     │              │ · SVG 景区地图    │
   │ · 设备控制       │              │ · 景点/路线/表演  │
   │ · 告警管理       │              │ · 故障上报+拍照   │
   │ · 数据历史       │              │ · 熄灯提醒        │
   └──────────────────┘              └──────────────────┘
```

**关键原则：一个后端，两个前端。** 两个前端通过 Vite proxy 各自代理 `/api` 请求到同一个 `localhost:3000`。后端无需任何架构变更。

## 四、游客端功能清单

### 4.1 地图主界面（核心）

- **SVG 手绘景区示意地图**，标注所有路灯位置
- **筛选叠加层**（类似 Forza Horizon 地图交互）：切换显示/隐藏拍照点、表演活动、游览路线
- **点击路灯** → 弹出该路灯附近的推荐内容
- **QR 码入口**：URL 携带路灯编号（`?lamp=lamp_003`），自动定位
- **手动定位**：输入路灯编号跳转
- **熄灯提醒横幅**：地图下方，"距离熄灯还有 XX 分钟，请合理安排游览"

### 4.2 夜景导览（游览路线）

- 推荐路线列表，每条路线标注沿途灯光亮点
- 地图上高亮显示选中路线

### 4.3 最佳拍照点

- 拍照点列表，含名称、描述、参考照片
- 地图上以相机图标标记

### 4.4 表演活动预告

- 景区表演列表：巡游、烟花、水幕电影等
- 每条含名称、时间、地点、倒计时、展示图
- 不支持路灯灯光秀（硬件仅能开关，无法 RGB 编程）

### 4.5 故障上报

- 表单：姓名 + 手机号 + 故障描述 + 拍照上传
- 接入后端已有告警系统（`alarms` 表）
- 上传成功提示，手机号仅记录不验证

### 4.6 不实现的功能

- ❌ 灯光秀控制（硬件能力不足）
- ❌ 110/120/119 紧急上报（游客自行拨打，景区不代报）
- ❌ 游客登录/注册（导览无需登录，上报时手工填身份信息）
- ❌ 实时 GPS 定位（SVG 地图不支持）

## 五、新增后端接口

| 方法 | 路径 | 说明 | 数据来源 |
|---|---|---|---|
| GET | `/api/scenic/routes` | 游览路线列表 | 静态 JSON |
| GET | `/api/scenic/spots` | 拍照点列表 | 静态 JSON |
| GET | `/api/scenic/events` | 表演活动列表 | 静态 JSON |
| GET | `/api/scenic/lamps` | 路灯位置列表（含坐标） | 静态 JSON / 复用 devices |
| POST | `/api/reports` | 故障上报（multipart，含图片） | 写入 alarms 表 |

## 七、技术决策记录

| # | 决策 | 原因 |
|---|---|---|
| 1 | 独立 Vue SPA 而非小程序/H5 | 复用现有技术栈，AI 辅助 5 天可交付；小程序语法差异大 |
| 2 | 景区数据后端返回静态 JSON | 与现有三层架构一致，后续换 DB 查询零前端改动 |
| 3 | SVG 手绘地图 | 免费、可控、免 API Key，适合学生演示 |
| 4 | QR 码定位而非 GPS | 路灯编号天然是定位锚点，扫码+手动双入口 |
| 5 | 图片上传走 HTTP 而非 MQTT | MQTT 不适配二进制大文件；Express 加 multer 一个路由即解决 |
| 6 | 故障上报接入已有 alarms 表 | 管理端告警页面直接展示，流程闭环 |
| 7 | 上报无需社交登录 | 填姓名+手机号即可，符合景区匿名使用场景 |
